# File Structure Viewer

React-приложение для отображения файловой структуры из `fs.json`.

Интерфейс похож на дерево файлов в VS Code: папки можно раскрывать, вложенные элементы отображаются под родительской папкой, файлы и папки отличаются иконками.

## Что умеет проект

- Загружает файловую структуру с сервера.
- Отображает дерево папок и файлов.
- Поддерживает любую глубину вложенности.
- Позволяет открывать папки кликом.
- Сворачивает открытую папку при повторном клике.
- Меняет URL при выборе папки.
- Подсвечивает выбранную папку.
- Реализует поиск по файлам и папкам.
- При поиске показывает найденные элементы вместе с родительскими папками.

## Запуск

Сначала должен быть запущен сервер, который отдает `fs.json`:

```bash
cd ../server
npm start
```

После этого запускается фронт:

```bash
cd ../front
npm install
npm run dev
```

Фронт будет доступен по адресу:

```text
http://127.0.0.1:5173/
```

## Структура проекта

```text
src/
  api/
    fileSystem.js
  components/
    FileExplorer.jsx
    FileItem.jsx
    Icons.jsx
  utils/
    tree.js
  App.jsx
  main.jsx
  styles.css
```

### `api/fileSystem.js`

Отвечает только за загрузку данных:

```js
fetchFileSystem()
```

Функция делает запрос к серверу:

```text
http://localhost:3001/fs.json
```

### `App.jsx`

Главный компонент приложения.

Он хранит:

- загруженное дерево файлов;
- текущий путь из URL;
- раскрытые папки;
- строку поиска;
- состояние загрузки.

Также `App` отвечает за смену URL при клике на папку.

Если пользователь повторно кликает по уже выбранной и раскрытой папке, `App` убирает ее из списка раскрытых папок.

### `components/FileExplorer.jsx`

Компонент верхнего уровня для дерева.

Он получает корневую папку, фильтрует ее элементы через поиск и передает каждый элемент в `FileItem`.

### `components/FileItem.jsx`

Главный компонент для отображения одного элемента дерева.

Он умеет отображать:

- файл;
- папку;
- вложенные элементы папки.

Именно здесь используется рекурсивная отрисовка.

### `utils/tree.js`

Файл с простыми функциями для работы с деревом:

- получение пути из URL;
- переход по URL;
- поиск папки по пути;
- сортировка элементов;
- фильтрация элементов при поиске;
- проверка, является ли элемент папкой.

## Формат данных

Приложение ожидает структуру примерно такого вида:

```json
{
  "root": {
    "movies": {
      "type": "folder",
      "children": {
        "Avengers": {
          "type": "folder",
          "children": {
            "Endgame": {
              "type": "file"
            }
          }
        }
      }
    }
  }
}
```

Папка определяется по `type: "folder"` или по наличию поля `children`.

Файл определяется как элемент без `children`.

## Паттерны

### Рекурсивный компонент

Основной паттерн проекта — рекурсивный компонент.

Он используется в `FileItem.jsx`.

Если элемент является файлом, компонент просто рисует строку файла.

Если элемент является папкой, компонент:

1. Рисует строку папки.
2. Получает дочерние элементы.
3. Для каждого дочернего элемента снова вызывает `FileItem`.

Пример:

```jsx
<FileItem
  name={childName}
  node={childNode}
  path={[...path, childName]}
  level={level + 1}
  selectedPath={selectedPath}
  expandedFolders={expandedFolders}
  searchQuery={searchQuery}
  onOpenFolder={onOpenFolder}
/>
```

Такой подход удобен для файловой структуры, потому что заранее неизвестно, сколько уровней вложенности будет в `fs.json`.

Если появится папка внутри папки внутри папки, код менять не нужно.

## Алгоритмы

### Обход дерева

Файловая структура является деревом.

Для отображения используется обход дерева сверху вниз:

1. Берем текущий элемент.
2. Проверяем, файл это или папка.
3. Если файл, отображаем его.
4. Если папка, отображаем ее.
5. Если папка раскрыта, повторяем те же действия для ее детей.

Так работает рекурсивный рендер в `FileItem.jsx`.

### Поиск в глубину

Поиск реализован в `tree.js` через функцию:

```js
matchesSearch(item, searchQuery)
```

Алгоритм:

1. Если строка поиска пустая, элемент подходит.
2. Если имя элемента содержит строку поиска, элемент подходит.
3. Если элемент является файлом и имя не совпало, элемент не подходит.
4. Если элемент является папкой, проверяем ее дочерние элементы.

Упрощенно:

```js
return getFolderItems(node).some(child => matchesSearch(child, query))
```

Это поиск в глубину, потому что функция спускается внутрь вложенных папок.

### Поиск папки по URL

Когда пользователь кликает по папке, URL меняется:

```text
/path/movies/Avengers
```

Функция `getFolderByPath` берет путь из URL и последовательно спускается по дереву:

```js
movies -> Avengers
```

Если папка найдена, она подсвечивается.

Если путь неправильный, приложение показывает сообщение:

```text
Папка не найдена.
```

### Сортировка

Элементы сортируются в `getFolderItems`.

Правила:

1. Сначала идут папки.
2. Потом идут файлы.
3. Внутри каждой группы элементы сортируются по имени.

Так дерево выглядит привычно для файлового менеджера.
